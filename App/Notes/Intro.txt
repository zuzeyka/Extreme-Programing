Extreme Programming (XP) - Экстремальное программирование
Вступление.
Управление программными проектами.
Управление проэктами - наука (с награждением научными степенями в области технических наук)
- Управление ресурсами
    = логистика / складовое управление
    = финансово/бюджетное управление
    = HR
- управление временем
- управление знаниями (проэкта)

Методология управления.
- waterfall
- agile
- scrum
- kanban
- XP

Линейная, жесткая, waterfall, классическая модель - набор задач, которые нужно выполнить одна за другой (последовательно) с возвозностью отдельных задач выполнять параллельно.
Главный визуальный инструмент - диаграмма Ганта
------------------------------------------------------------------------------------------->время
Составления ТЗ
----------------
                Проэктирование UI
                -----------------
                Проэктирование API
                --------------------------------
                                Тестирование UI
                                -----------------
                                            Тестирование API
                                            -----------------
Плюсы: наглядность, возможность спрогнозировать результат, управляемость (в любой момент времени понятно что должно быть завершено, что в процессе, итд)
Минусы: наличие критического пути (определенные этапы являются критическими и увеличение временени выполнения на них увеличивает продолжительность всего проекта с возможным простоем других задач),
долгое время на разработку готовой для продакшен тестовой версии, сложность внесения изменения в структуру проекта
Гибкие: agile, итеративные, водоворотные, спиральные - модели управления с постоянными повторениями однотипных итираций
Планирование - Проэктирование - Тестирование - Внедрение
В спиральных моделях имеется в виду, что каждая следующая итерация имет больший маштаб
Главный визуальный инструмент - scrum-board

Экстремальное программирование - разновидность гибких моделей спиральных моделей, который направлен на абсолютизацию отдельных моментов, особенно тех, которые противоречат один другому, например, максимально быстрый первый выпуск --- нормализованный рабочий график (день/неделя)
